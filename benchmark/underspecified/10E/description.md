We've got two people building separate pieces of the same software project at the same time, plus a third person who stitches everything together at the end. Both builders need to compile against one shared core library, but it can only be used by one of them at a time, so they have to take turns with it. The first builder also looks after a shared file of type definitions that the second builder relies on when compiling; that file can only be touched by one person at once, with the first writing new definitions and the second reading them. When the first builder changes those definitions, it lets the second know. Here's the tricky part: if those types get updated after the second builder already compiled against the older version, the second builder has to throw away that build and redo it with the new types. The two work concurrently, so who grabs what first isn't fixed. Once both have finished their builds and told the third person they're done, that person links everything into the final artifact. We're finished when both modules are compiled and the final artifact is linked.
